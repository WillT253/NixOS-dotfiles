# Contributing to WillT253/NixOS-Dotfiles

I don't know why you would want to put yourself through this, but if you accept the fact that you'll spend most of your time questioning how code can be *this* bad, here's some guidelines.

## "I've found a bug / issue / fault"

Opening an issue with a clear and descriptive title and a full rundown of what you did to find/cause it is the minimum. Optionally, if you have found what is causing the issue, quote the file and line-number(s) that contain the issue. If you have a fix, PRs are welcome, though it may take some time to merge this into the repo. 

## "I have a suggestion"

If the suggestion is something that changes an option that has already been set (e.g. changing a default keybind), it is unlikely that this will make it into the repo. Still, I welcome suggestions of this sort as I am always looking to streamline my setup.

If your suggestion is for a new feature (programs, keybinds, themes, etc.) then I will add it if I think it would benefit more than half of the users. I am working on modularising the config files to allow for your personal preferences should you want to change the preconfigured settings though, so if I veto a request you may be able to implement it on your configuration soon anyway.

If your suggestion is a code change that doesn't affect themes, keybinds, etc. (e.g. rewriting some part of a file more optimally) make a PR with your change(s) and open an issue.

## "I have a question"

I would prefer if questions are **not** opened as issues as that will cause clutter. However, you can ask questions in the Q&A section of the Discussions tab.
